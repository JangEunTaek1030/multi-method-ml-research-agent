## Key Design Decision

This project will not start directly from LLM APIs.

It will first build traditional machine learning baselines, then progressively add deep learning, Transformer, BERT, RAG, and LLM modules.

Reasons:

1. Traditional ML provides interpretable and stable baselines.
2. Deep learning introduces representation learning.
3. Transformer implementation helps understand modern NLP models.
4. BERT fine-tuning connects pretraining with downstream tasks.
5. RAG connects retrieval with generation.
6. LLM integration turns the system into a practical research assistant.

## Interview Narrative

This project demonstrates a complete learning and engineering path from traditional machine learning to modern LLM applications.

Instead of only using an LLM API, the project implements and compares multiple modeling approaches, including TF-IDF based traditional ML models, PyTorch neural networks, a Mini Transformer from scratch, BERT fine-tuning, and RAG-based document question answering.
